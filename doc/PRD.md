# Product Requirements

## Product
Digital Banking MVP for a hackathon build.

## Delivery Model
- Phase 1: 6-hour core build
- Phase 2: 3-hour important expansion
- Phase 3: 1-hour nice-to-have pass

## Phase 1 Goals
- User signup, login, logout, and session persistence
- Dashboard with account summary, balance overview, and recent transactions
- Transactions page with basic search/filter and detail view
- Internal transfer flow with confirmation and transfer history
- Realtime notifications
- Basic admin monitoring dashboard
- Core UI polish and essential testing

## Phase 2 Goals
- Password reset
- MFA or OAuth
- Account details
- Statements download
- External account linking
- External transfers
- Scheduled transfers
- Bill payments
- Stronger admin alerts
- Additional security handling

## Phase 3 Goals
- Reserved for later nice-to-have features to be provided by the user

## Demo Flow
1. User signs up
2. User logs in
3. Dashboard shows accounts
4. User views transaction history
5. User transfers funds
6. Admin sees transaction instantly
